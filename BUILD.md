# dunamismax.com · Build & Deploy Runbook

`dunamismax.com` is a fully static site generated by a small Python build
tool. The output is a directory of HTML, CSS, JS, SVG, and an RSS feed that
Caddy serves as-is on a single Ubuntu box. There is no application
runtime, no database, no background process, and no third-party JavaScript.

## What's live now

- `dist/` builds cleanly from a fresh clone (`python3 scripts/build.py build`).
- The home page, projects index, blog index, blog post detail, about,
  contact, RSS feed, robots, manifest, and 404 all render.
- The compiled TypeScript theme toggle preserves the original behaviour and
  the `dunamismax-theme` localStorage key. The site is fully readable with
  JavaScript disabled.
- Homepage HTML + CSS + JS is roughly **8 KB gzipped** (target < 50 KB).
- `python3 scripts/build.py check` validates content, walks every internal
  link in `dist/`, and runs `tsc --noEmit`.

#### Production cutover sequence

```sh
sudo systemctl stop dunamismax-web.service
sudo systemctl disable dunamismax-web.service
sudo rm -f /etc/systemd/system/dunamismax-web.service
sudo systemctl daemon-reload

cd /home/sawyer/github/dunamismax.com
git pull --ff-only
python3 scripts/build.py build

sudo cp Caddyfile /etc/caddy/Caddyfile
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy

# Confirm the static surface
curl -sI https://dunamismax.com/ | head
curl -s https://dunamismax.com/feed.xml | head
```

After the static deploy is verified, the local `storage/*.sqlite*` files
can be archived and removed.

## Toolchain

The repo targets Python 3.11+ and TypeScript via `tsc`. There are no
runtime npm packages and one optional Python package (`markdown`) for
blog post bodies.

```sh
# Ubuntu
sudo apt-get install -y python3 python3-markdown

# tsc (one-shot)
npm install -g typescript    # or `mise install` if mise is in use
```

## Local commands

```sh
python3 scripts/build.py build   # produce dist/
python3 scripts/build.py serve   # http://127.0.0.1:8000 with rebuild-on-change
python3 scripts/build.py check   # tsc --noEmit + content + link walk
python3 scripts/build.py clean   # rm -rf dist/
```

## Production shape

The deployment target is one Ubuntu box: Caddy serves `dist/` directly
behind Cloudflare. The redeploy flow is:

```sh
cd /home/sawyer/github/dunamismax.com
git pull --ff-only
python3 scripts/build.py build
sudo systemctl reload caddy
```

The `Caddyfile` block lives at `/etc/caddy/Caddyfile`:

```caddyfile
www.dunamismax.com {
    redir https://dunamismax.com{uri} permanent
}

dunamismax.com {
    encode zstd gzip
    root * /home/sawyer/github/dunamismax.com/dist
    file_server

    handle_errors {
        @404 expression {http.error.status_code} == 404
        rewrite @404 /404.html
        file_server
    }

    header {
        X-Content-Type-Options "nosniff"
        X-Frame-Options "DENY"
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        Referrer-Policy "strict-origin-when-cross-origin"
        X-XSS-Protection "0"
        Permissions-Policy "camera=(), microphone=(), geolocation=()"
        -Server
    }
}
```

This block is current for the static-site cutover (Phase 6). Until then,
the live `Caddyfile` in this repo still reverse-proxies to Puma.
